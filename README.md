# texture22 - 第２６回 レンダリング画像をテクスチャに使う サンプルプログラム

## 1. 概要

このプログラムは、OpenGL における「テクスチャマッピング (Texture Mapping)」の基礎を学ぶための、学生向けのサンプルプログラムです。本プログラムは、以下のブログ記事のサンプルプログラムです。

- [第２６回 レンダリング画像をテクスチャに使う](https://tokoik.github.io/blog/%E3%83%86%E3%82%AF%E3%82%B9%E3%83%81%E3%83%A3%E5%85%A5%E9%96%80/2005/09/23/texture.html)

![レンダリング画像をテクスチャに使う](https://tokoik.github.io/blog/assets/images/texture/texture22b.webp)

## 2. ビルド方法

このプログラムは CMake を用いてビルドを構成しています。各プラットフォームごとの手順は以下の通りです。なお、プログラムをビルドするためのバイナリディレクトリは、バージョン管理ファイル（.gitignore）の設定に合わせて build という名前にします。

### 2.1 Windows (Visual Studio 2022 の場合)

1. コマンドプロンプトまたは PowerShell を開き、このプロジェクトのディレクトリに移動します。
2. 以下のコマンドを実行してビルドディレクトリを作成し、CMake で構成を行います。

   ```bat
   mkdir build
   cd build
   cmake .. -G "Visual Studio 17 2022"
   ```

3. 生成された build フォルダ内の texture22.sln を Visual Studio で開きます。
4. ソリューションエクスプローラーで texture22 プロジェクトを右クリックし、「スタートアップ プロジェクトに設定」を選択します。
5. 「ローカル Windows デバッガー」をクリックするか、F5 キーを押してビルドおよび実行します。

### 2.2 macOS (Xcode の場合)

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。
2. 以下のコマンドを実行してビルドディレクトリを作成し、Xcode 用のプロジェクトを生成します。

   ```sh
   mkdir build
   cd build
   cmake .. -G Xcode
   ```

3. 生成された build/texture22.xcodeproj を Xcode で開きます。
4. 左上のスキーム選択（再生ボタンの横）が texture22 になっていることを確認します。
5. 「Run」ボタン（再生ボタン）をクリックするか、Command + R を押してビルドおよび実行します。

### 2.3 Ubuntu Linux

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。
2. 必要なパッケージ（freeglut3-dev や pkg-config など）がインストールされていることを確認し、以下のコマンドでビルドします。

   ```sh
   mkdir build
   cd build
   cmake ..
   make
   ```

## 3. 使い方

### 3.1 プログラムの起動方法

各OSとも、ビルド後に生成されるバイナリディレクトリ (build) やそのサブフォルダから起動します。（※ CMake の設定により、Windows や Xcode では Debug などのフォルダ下に実行ファイルが置かれることがあります）

- **Windows**

  Visual Studio 上で「ローカル Windows デバッガー」をクリックして実行するか、またはコマンドプロンプトから以下のコマンドで起動します。

  ```cmd
  cd build\Debug
  texture22.exe
  ```

- **macOS**

  Xcode 上で左上の「Run（再生ボタン）」をクリックするのが楽です。これにより texture22.app アプリケーションバンドルとして自動的に実行されます。アプリケーションバンドルを直接起動するなら、Finder から build/Debug/texture22.app をダブルクリックするか、ターミナルから open build/Debug/texture22.app を実行します (この場合はエラーメッセージ等が表示されません)。

- **Ubuntu Linux**

  ターミナルから以下のコマンドで実行ファイル（バイナリ）を直接起動します。

  ```sh
  cd build
  ./texture22
  ```

### 3.2 操作方法

- **マウスの左ボタンでドラッグ**

  空間全体または物体を回転させることができます。周囲の無数の立方体（星）が、動的にレンダリングされて中央のティーポットに環境マッピング（キューブマッピング）として映り込んでいる様子が確認できます。

- **キーボードの q, Q または ESC キー**

  プログラムを終了します。

## 4. 解説

プログラムのソースコード（主に main.cpp）をもとに、このプログラム内で行われている処理手順を解説します。このプログラムでは、プログラムの実行中に動的に画像をレンダリングし、それをキューブマッピングのテクスチャとして利用することで、周囲の環境がリアルタイムに反射する表現を実現しています。

### 4.1 テクスチャ用画像の動的生成（オフスクリーンレンダリングの代替）

テクスチャ用の画像を生成するには、通常は画面に表示されないオフスクリーンバッファ（pbuffer や Framebuffer Object 等）を使用します。しかし本プログラムでは簡便のため、ダブルバッファリングにおける**バックバッファ**を利用しています。`glutSwapBuffers()` を呼び出すまでの間にテクスチャ用のレンダリングとメイン画面のレンダリングの両方を行ってしまうことで、テクスチャ生成過程を画面に表示させずに済ませています。

### 4.2 テクスチャと環境の初期化 (`init()` 関数)

1. **空のテクスチャの確保**
   キューブマッピング用の6面のターゲットそれぞれに対して、画像データとして 0 ( `NULL` ) を渡して `glTexImage2D()` を呼び出し、必要なサイズのテクスチャメモリだけを確保します。
   
2. **キューブマッピングの設定**
   テクスチャ座標の自動生成モードを `GL_REFLECTION_MAP` に設定し、物体の表面で反射するようなテクスチャ座標が生成されるようにします。

3. **周囲の星（立方体）の生成**
   キューブマップの環境として映り込む「星」の代わりとして、空間内にランダムに配置された無数の立方体を生成し、ディスプレイリスト (`stars`) に登録しておきます。

### 4.3 動的テクスチャの生成 (`display()` 関数の前半)

`display()` 関数が呼ばれると、まずテクスチャ用の6面の画像をレンダリングします。

1. **6方向のレンダリング**
   `target` 配列に定義された各面について、画面の端をビューポート（`glViewport()`）として割り当てます。そして `gluLookAt()` でカメラをその面が表す方向（上下左右前後）に向け、ディスプレイリストに登録された星（`stars`）を描画します。

2. **テクスチャメモリへのコピー**
   レンダリングが1面終わるごとに `glCopyTexSubImage2D()` を呼び出し、バックバッファに描画された画像をテクスチャメモリの該当する面（`GL_TEXTURE_CUBE_MAP_POSITIVE_X` など）に直接コピーします。

3. **テクスチャの反転**
   こうして得られたテクスチャをそのままキューブマッピングに使うと裏返しになってしまうため、テクスチャ行列 (`GL_TEXTURE`) を用いて `glScaled(`-1.0, -1.0, 1.0 `)` と設定し、テクスチャ座標の x, y を反転させます。

### 4.4 最終的な画像のレンダリング (`display()` 関数の後半)

テクスチャの準備が完了したら、いよいよ最終的な画像をレンダリングします。

1. **画面のクリアとビューポートの再設定**
   テクスチャ作成用に描画されたバックバッファの内容は不要になるため、`glClear()` で画面全体をクリアします。そして `glViewport(` 0, 0, `width`, `height` `)` でウィンドウ全体をビューポートに設定し直します。

2. **シーンの描画 (`scene()` 関数)**
   視点の移動やトラックボールの回転を適用した後、`scene()` 関数で中央のティーポット（または球体）を描画します。ここでは `GL_TEXTURE_CUBE_MAP` を有効にし、さらに `GL_TEXTURE_GEN_S`, `GL_TEXTURE_GEN_T`, `GL_TEXTURE_GEN_R` を有効にしてテクスチャ座標を自動生成させます。これにより、前半で生成した周囲の星のテクスチャがティーポットに環境マッピングとして映り込みます。

3. **画面の更新**
   最後に `glutSwapBuffers()` を呼び出すことで、バックバッファに描画された完成画像がフロントバッファと入れ替わり、画面に表示されます。
